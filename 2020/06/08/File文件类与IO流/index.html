File文件类与IO流

File文件类与I/O流

  1. File文件类
    1.1 基本构造方法

    • File(String pathName):通过将给定路径名字符串转换为抽象路径名来创建一个新 File 实例。

    • File(File parent, String child):根据 父抽象路径名和 子路径名字符串创建一个新 File 实例。

    • File(String parent, String child): 根据 父路径名字符串和 子路径名字符串创建一个新 File 实例。

    • 注意要点

      • 一个File对象代表硬盘中实际存在的一个文件或目录
      • File类构造方法不会检验文件或文件夹是否真实存在，因此无论该路径下是否存在文件或目录，都不影响File对象的创建

      1.2 获取功能的方法

    • public String getAbsolutePath():返回此抽象路径名的绝对路径名字符串。

    • public String getPath():将此抽象路径名转换为一个路径名字符串。

    • public String getName():返回由此抽象路径名表示的文件或目录的名称。

    • public long length(): 返回由此抽象路径名表示的文件的长度。

    • 注意要点

      - 路径(文件夹)使用`getName()`只能返回当前一级路径名，使用`getPath()`才能返回**当前路径的全路径**
      • 路径不区分大小写
      • 路径中的文件名分隔符windows使用反斜杠\，反斜杠是转义字符，两个反斜杠\\代表一个普通斜杠
      • 文件使用length()返回文件所占空间的字节大小，目录返回的则不是所占空间的字节大小

      1.3 判断功能的方法

      • public boolean exists(): 测试此抽象路径名表示的文件或目录是否存在。

      • public boolean isFile():测试此抽象路径名表示的文件是否是一个标准文件。

      • public boolean isDirectory():测试此抽象路径名表示的文件是否是一个目录。

      • 注意要点

    • isFile()和isDirectory()会判断文件或目录是否存在，不存在返回false。

      1.4 创建和删除功能的方法

      • public boolean createNewFile():当文件不存在时创建一个新文件并返回true否则返回false
      • public boolean delete(): 删除此抽象路径名表示的文件或目录。
      • public boolean mkdir():创建此抽象路径名指定的目录。
      • public boolean mkdirs(): 创建此抽象路径名指定的目录，包括所有必需但不存在的父目录。
      • 注意要点
      • 使用delete()删除目录，若目录下包含文件或子目录(无论该子目录是否为空)，则无法删除返回false
      • mkdir()只能创建一级目录，若父目录不存在，则无法创建返回false
      • mkdir()可以创建多级目录，即可以创建不存在的父目录

      1.5 遍历目录

      • public String[] list():返回一个字符串数组，这些字符串指定此抽象路径名表示的目录中的文件和目录。
      • public File[] listFiles():返回一个抽象路径名数组，这些路径名表示此抽象路径名表示的目录中的文件。
      • 注意要点
      • File构建的必须是目录且指定目录必须存在，否则会报NullPointerException空指针异常
